\section{Project Overview}
This project implements a Meta Quest XR application in Unity with a login system, project listing, dynamic floor selection, backend API integration, XR-compatible navigation, and a polished world-space user interface. The experience is designed as a compact Quest-style product flow rather than a generic desktop form.

\section{Assignment Requirements Covered}
\begin{itemize}
    \item Login screen with username, password, login button, status, and toast feedback.
    \item Backend API integration for login, project listing, and floor listing.
    \item Project list rendered dynamically from API response data.
    \item Floor options rendered dynamically based on the selected project.
    \item Forward and backward navigation across login, project, floor, and summary screens.
    \item XR world-space UI suitable for controller ray interaction.
    \item Toast notifications for success, failure, selected project, and selected floor.
    \item Optional dark/light visual theme support for final UI polish.
    \item Runtime XR usability support including comfortable panel placement, a draggable panel handle, and an in-world keyboard fallback.
    \item Testing path prepared for Meta XR Simulator, Quest headset Play Mode, and Android APK build.
    \item Organized Unity scripts and a Vercel-ready backend folder.
\end{itemize}

\section{System Architecture}
The solution is split into two main parts:

\begin{itemize}
    \item \textbf{Unity XR Frontend}: Handles world-space UI, session state, navigation, toast feedback, runtime XR interaction support, and API calls through UnityWebRequest.
    \item \textbf{Vercel Backend}: Provides serverless API routes using a minimal Next.js application with hardcoded assignment data.
\end{itemize}

The Unity application calls the deployed backend URL configured in \path{Assets/Scripts/API/APIConfig.cs}. The application stores the login token, user, selected project, and selected floor in a session manager.

}
\end{verbatim}

Success response:
\begin{verbatim}
{
  "success": true,
  "token": "demo_jwt_token_123",
  "user": {
    "id": 1,
    "name": "Test User"
  }
}
\end{verbatim}

\subsection*{GET /api/projects}
Returns four selectable projects: Project A, Project B, Project C, and Project D.

\subsection*{GET /api/projects/\{id\}/floors}
Returns floors for the selected project. Each project has different floor data to demonstrate dynamic dropdown behavior.

\section{Unity XR Implementation}
The Unity project uses a single-scene architecture for reliability and speed. The scene contains a Meta Quest-style world-space dashboard, Inspector-wired UI references, button navigation logic, backend API calls, and runtime XR interaction safeguards.

Core script groups:
\begin{itemize}
    \item \texttt{Assets/Scripts/API}: API configuration, models, login API, and project API.
    \item \texttt{Assets/Scripts/Managers}: Session and navigation state.
    \item \texttt{Assets/Scripts/UI}: Login, project list, floor dropdown, toast, controller feedback, and scene UI bindings.
\end{itemize}

Additional runtime support is handled by \texttt{XrRuntimeStabilizer.cs}. It keeps the UI at a comfortable Quest viewing distance, prevents unintended falling from locomotion/gravity during this assignment flow, configures XR UI input, adds a draggable panel handle, and provides an in-world keyboard fallback when the system keyboard is unavailable.

\section{Meta Quest UI/UX Design Approach}
The interface is designed to feel closer to a Quest-native spatial UI. It uses floating panels, high contrast typography, rounded buttons, blue selected states, hover/press feedback, accent rails, status pills, and world-space toast notifications. The UI is readable at VR distance and avoids a flat desktop form layout.

The visual direction uses an Inter/Roboto-style modern sans-serif approach, with TextMeshPro's default font as a safe project fallback. The world-space dashboard is positioned in front of the XR origin at a comfortable viewing distance for Quest testing. A small theme controller is included so the interface can support dark and light presentation modes without changing the backend or navigation logic. The login inputs also include a demo credential fallback so the full flow can be demonstrated reliably in simulator and APK recordings.

\section{Testing Checklist}
\begin{itemize}
    \item Correct login opens the project list.
    \item Wrong login shows an invalid credentials toast.
    \item Project list loads from the backend API.
    \item Selecting a project highlights it, and Continue loads only that project's floors.
    \item Selecting a floor highlights the selected option.
    \item Summary screen shows user, project, and floor.
    \item Back navigation works.
    \item XR ray interaction works with controller input.
    \item Toast notifications fade correctly.
    \item XR Origin is positioned at the scene origin and the world-space dashboard appears comfortably in front of the player.
    \item In-world keyboard appears for input fields and demo credentials can be filled without relying on the platform keyboard.
    \item Dashboard handle can be used to reposition the world-space UI if needed.
    \item Backend tests and production build pass.
    \item APK installs and opens on a Meta Quest device or simulator.
\end{itemize}

The application supports testing through Meta XR Simulator and Meta Quest headset builds. Editor validation can be performed with Meta XR Simulator, while final device validation should use a Meta Quest 2, Quest 3, or Quest Pro APK install. A video recording is submitted separately to show the simulator/headset flow.

\section{Build and Installation Instructions}
\begin{enumerate}
    \item Backend is deployed to Vercel at \url{https://visnet-assignment.vercel.app}.
    \item Confirm \texttt{APIConfig.cs} points to the deployed Vercel URL.
    \item Open the Unity project in Unity 6000.0.70f1 or compatible Unity 6 version.
    \item Open \texttt{Assets/Scenes/MainScene.unity}.
    \item Verify \texttt{MainScene.unity} contains the world-space panels and assigned \texttt{AppManager} references.
    \item Switch build target to Android.
    \item Use OpenXR, IL2CPP, ARM64, and ASTC texture compression.
    \item Build an APK for Meta Quest and install it on Quest 2, Quest 3, or Quest Pro for final testing.
    \item Record the login, project selection, floor selection, and summary screen flow for submission evidence.
\end{enumerate}
